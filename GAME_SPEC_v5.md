# AI2037 ゲーム仕様書 v5.0（設計レビュー統合版）

v4.0 からの主要変更点：
- UI設計を「議長端末（Chairman's Terminal）」統一UIに刷新
- Base/Run のモード分離を廃止し、単一画面の状態変化で表現
- Run フェーズのコアメカニクスを新規定義（リスク蓄積・ステップ制）
- Act境界を自動処理から「決算会議」形式に変更（プレイヤー参加型）
- 画面構成を再設計（フル画面7＋オーバーレイ5）
- セッション時間を再計算し整合性を確保
- タイトルと世界観の整合性について課題を記録

---

## 1. コンセプト

**一行要約：**
「善意の組織がなぜ破滅するかを体験する、AI開発シミュレーション」

**テーマ：**
AI開発の加速と、供給制約・制度・社会反応・倫理リスクの同時進行

**体験の核：**
「自分は正しく選んだはずなのに、なぜこうなった」と思わせること

**プラットフォーム：** モバイル（買い切り）
**1セッション：** 16〜22分
**1キャンペーン：** 6〜8 Act（1 Act = 1セッション）
**周回前提：** 2〜4周で全体像が見える設計
**全体所要：** 1キャンペーン約2〜2.5時間 / 全体像把握まで約5〜10時間

---

## 2. タイトル「AI2037」に関する課題

「AI」＋「近未来年号」で内容を即座に伝達できる強みがある一方、以下の未解決課題がある：

- 仕様上「2037年」がゲーム内のどこにも接地していない
- 周回設計が「別惑星・別文明への輪廻」であり、地球の年号に縛られない構造と矛盾する

**対応方針（要決定）：**
1. 1周目を2037年の地球に固定し、周回で「2037年」の意味が変容する構造にする
2. 年号を外し、文明横断に耐えるタイトルに変更する

## 3. ゲーム構造

```text
1 Act（= 1セッション、16〜22分）

Base（4〜6分）
 ├─ 資源・人材・制度の提案カード（8〜12枚）
 ├─ 政策イベントカード（Base中に3〜5回）
 └─ Run準備確認カード
       │
Run（10〜14分）
 ├─ 研究ステップ（8〜12ステップ）
 ├─ 速度↔安全の逐次判断
 ├─ リスク蓄積 → 事故イベント
 └─ Run完了
       │
清算（1〜2分）
 ├─ 進行度・資源変動の報告
 ├─ 顧問コメント
 └─ AI段位更新判定
       │
Act境界：決算会議（2〜3分）
 ├─ 段階1: 組織状況報告
 ├─ 段階2: 顧問所見
 ├─ 段階3: Key Event（Act固有のナラティブイベント）
 ├─ 段階4: 議長方針宣言
 └─（裏側処理）不可逆スコア集計・RouteProfile再計算・分岐判定
       │
次 Act へ（最終 Act の場合 → エンディング）
```

**コアサイクル：** Base → Run → 清算 → Act境界 を1セッションで1巡

## 4. 状態変数（合計12個）

### 表示変数（UI表示、6個）

| 変数 | 役割 |
|------|------|
| 資金 | 全体の行動制約。全投資の源泉 |
| 電力 | 計算資源の前提条件 |
| 計算資源（マシン） | Run進行の燃料。毎ステップ消費 |
| 人材 | 配置・効率・離職リスク |
| 制度 | 監査・法務・手続きの成熟度 |
| AI段位 | Tier表示。段位で解放・難度・不穏が変質 |

### 資源チェーン

```
資金 ──投資──→ 電力 ──投資──→ 計算資源
  │                                 │
  │          ┌──────────┐           │
  └─維持費──→│  人材    │←─配置先─→ Runの効率
              └──────────┘
                  ↕
              ┌──────────┐
              │  制度    │→ 長期安定性
              └──────────┘
```

- 電力がないと計算資源は増やせない
- 人材不足はRun効率低下＋離職イベントを誘発
- 制度投資は短期的にRunを遅らせるが、後半Actの崩壊を防ぐ

### 非表示変数（数値非公開、2個）

| 変数 | 役割 |
|------|------|
| 倫理（Ethics） | 補償・透明性・弱者配慮の姿勢 |
| 自己欺瞞（SelfDeception） | 都合の悪い兆候を"見ない"組織文化 |

**非表示変数の可視化方針：**
数値は出さない。代わりに**症状**で示す。

| 内部状態 | 症状表現 |
|----------|---------|
| 倫理低下 | ニューストーン悪化、外部抗議、顧問の沈黙 |
| 自己欺瞞上昇 | 会議ログの婉曲化、リスク文言の軟化、顧問が耳障りの良い報告しかしなくなる |
| 自己欺瞞高＋倫理低 | 新しい選択肢が生える（「却下する」「報告を書き換える」等。選択肢の存在自体が腐敗の証拠） |

### 不可逆スコア（内部蓄積、4個）

| スコア | 蓄積条件 |
|--------|---------|
| Mercy | 補償・救済・弱者配慮の選択 |
| Dominion | 支配・独占・競合排除の選択 |
| Reckless | 速度優先・検証省略・リスク無視の選択 |
| Truth | 公開・透明性・内部告発容認の選択 |

**用途：** Act分岐判定、会話差分、イベント抽選重み、エンディング条件

## 5. Base（準備フェーズ）

### 基本仕様

| 項目 | 値 |
|------|-----|
| 目的 | ボトルネックを解消し、Runの条件を整える |
| 所要時間 | 4〜6分 |
| カード枚数 | 8〜12枚 |
| 1カードの判断時間 | 約25〜35秒 |
| 端末状態 | Base状態（標準表示） |
| 中断 | カード間でいつでも可能 |

### カード種別と配分

| カード種別 | 枚数/Base | 提出者 | 内容 |
|-----------|----------|--------|------|
| 資源カード | 2〜3枚 | 運営顧問 | 予算配分、インフラ投資、電力確保 |
| 人材カード | 1〜2枚 | 人事顧問 | 人員配置（研究/安全/制度/広報）、採用、離職対応 |
| 制度カード | 1〜2枚 | 法務顧問 | 監査体制、内部規程、第三者検証の導入 |
| 政策イベント | 3〜5枚 | （外部発生） | 規制変更、世論、競合動向、メディア報道 |

### 人材配置先と効果

| 配置先 | 効果 | 不足時のリスク |
|--------|------|--------------|
| 研究 | Run進行速度が上昇 | 進行が遅くなる |
| 安全 | Runのリスク閾値が上昇（事故しにくい） | リスク閾値低下、少しの無理で事故 |
| 制度 | 制度値の自然上昇、後半の制度イベント緩和 | 制度の自然劣化、規制違反イベント増加 |
| 広報 | 倫理（Ethics）への好影響、世論イベント緩和 | 社会的信頼低下、抗議・報道イベント増加 |

### カード設計原則

- 各カードは状況説明＋2〜3択
- 選択肢は過去の行動履歴・自己欺瞞レベルで動的に増減
- 各選択は必ず以下を動かす：
  - 表示変数 1〜2個
  - 非表示変数 0〜2個
  - 不可逆スコア 0〜1個

### Baseフェーズの流れ

```
Base開始
  │
  ├─ カード1: 前Actの引き継ぎ報告（固定・情報提供のみ）
  ├─ カード2〜3: 資源カード
  ├─ カード4: 政策イベント（1回目）
  ├─ カード5〜6: 人材・制度カード
  ├─ カード7: 政策イベント（2回目）
  ├─ カード8〜9: 資源 or 政策イベント（状況により変動）
  ├─ カード10: Run準備確認カード（固定）
  │    「研究チームの準備が整いました。Runを開始しますか？」
  │     [開始する] → Runフェーズへ
  │     [もう少し準備する] → 追加カード1〜2枚
  │
  └─ → Runフェーズへ
```

Run準備確認カードがBaseからRunへの唯一の明示的遷移点。議長端末のUI自体は変わらず、届くカードの文脈が変わる。

## 6. Run（研究フェーズ）

### 基本仕様

| 項目 | 値 |
|------|-----|
| 目的 | AI段位を進行させる。速度↔安全のトレードオフを体験する |
| 所要時間 | 10〜14分 |
| ステップ数 | 8〜12ステップ（AI段位が高いほど増加） |
| 1ステップの判断時間 | 約60〜90秒 |
| 端末状態 | Run状態（進行バー＋データ流表示を追加） |
| 中断 | ステップ間で可能（自動セーブ） |

### 3つの内部カウンター

| カウンター | 初期値 | 可視性 | 役割 |
|-----------|--------|--------|------|
| 進行度（Progress） | 0 | ✅ 表示（進行バー） | AI段位進行の燃料 |
| リスク蓄積（Risk） | 0 | ❌ 非表示 | 事故発生の判定に使用 |
| 計算資源（Compute） | Base終了時の値 | ✅ 表示（消費バー） | 毎ステップ消費、枯渇でRun強制終了 |

### 速度↔安全のトレードオフ

各ステップの選択肢は以下の基本パターンに沿う：

| 選択傾向 | 進行度 | リスク | 計算消費 | 不可逆スコア |
|---------|--------|--------|---------|------------|
| 加速 | +3 | +2〜3 | 標準 | Reckless+1 |
| 標準 | +2 | +1 | 標準 | なし |
| 慎重 | +1 | +0 or -1 | 標準 | なし |
| 検証 | +0 | -2 | 追加消費 | Truth+1 |

加速は常に魅力的（進行度3倍）だが、リスクが見えない。プレイヤーはリスク値を直接知ることはできない。

### リスク蓄積と事故

リスク閾値（AI段位により変動）：

| AI段位 | リスク閾値 |
|--------|----------|
| Tier 1〜2 | 15（余裕がある） |
| Tier 3 | 12 |
| Tier 4 | 9 |
| Tier 5+ | 6（ほぼ余裕なし） |

| リスク蓄積 | 発生する事象 |
|-----------|------------|
| 閾値の50%到達 | 予兆カード（自己欺瞞低→警告 / 高→「問題ありません」） |
| 閾値到達 | 軽微事故: 計算資源消失、1ステップ消失 |
| 閾値の150% | 重大事故: 人材-1〜2、倫理-2、Run強制中断 |
| 閾値の200% | 壊滅的事故: 大量の資源喪失、即Act境界へ。特殊エンディングに直結しうる |

事故は確率ではなく蓄積で発生する。プレイヤーが何度も加速を選べば必ず事故が起きる。

### AI段位（Tier）進行

| 遷移 | 必要累計進行度 | 目安Run数 |
|------|-------------|----------|
| Tier 1→2 | 20 | 約2Run |
| Tier 2→3 | 50 | 約3Run |
| Tier 3→4 | 100 | 約4Run |
| Tier 4→5 | 180 | 約5Run |
| Tier 5→5+ | 到達不可 | エンディング条件 |

1キャンペーン（6〜8Run）でTier 4〜5到達が標準。全加速プレイなら6RunでTier 5可能だが事故リスク極大。

### ステップ種別（8種）

| 種別 | 出現率 | 報告者 | 状況 |
|------|--------|--------|------|
| ① 通常進行 | 25% | 研究リーダー | 通常の進行判断 |
| ② 異常検知 | 15% | 安全チーム | 予期しない出力パターンの検出 |
| ③ 突破口 | 10% | 研究チーム | 高リスクなショートカットの発見 |
| ④ 資源逼迫 | 10% | 運営 | 計算資源の想定超過消費 |
| ⑤ 倫理的問題 | 15% | 法務/外部 | 学習データの権利・バイアス問題 |
| ⑥ 外圧 | 10% | 広報/外部 | 競合動向、投資家圧力 |
| ⑦ 安全警告 | 10% | 安全システム | モニタリングの異常値記録 |
| ⑧ チーム問題 | 5% | チーム個人 | 倫理的懸念による異議申し立て |

### Runフェーズの流れ

```
Run開始（端末がRun状態に変化、進行バー表示）
  │
  ├─ Step 1: 通常進行（固定。ウォームアップ）
  ├─ Step 2〜4: ランダム（Act/Tierに応じて抽選）
  │
  ├─ ★ 中間報告カード（固定・情報提供のみ）
  │    「研究は[X]%進行。計算資源残量[Y]」
  │
  ├─ Step 5〜8: ランダム（後半は⑤⑦⑧の比率が上昇）
  ├─ (高Tierの場合) Step 9〜12: 追加ステップ
  │
  ├─ ★ リスク閾値超過 → 事故カード割り込み
  ├─ ★ 計算資源枯渇 → Run強制終了
  │
  └─ Run完了 → 清算フェーズへ

Runの終了条件：
  1. 全ステップ完了（正常完了）
  2. 計算資源枯渇（早期終了）
  3. 壊滅的事故発生（緊急中断）
```

### 清算フェーズ（1〜2分）

Run直後に表示される結果報告カード。

表示内容：
- 今回の進行度と累計（次Tierまでの距離）
- 計算資源の消費量
- Run中の重要判断の振り返り（2〜3件）
- 顧問コメント（1〜2人が一言）
- Tier上昇があればここで宣言

自己欺瞞フィルター：SelfDeception高→事故や問題が報告から省略される。

## 7. Act境界（決算会議フェーズ）

### 基本仕様

| 項目 | 値 |
|------|-----|
| 目的 | このActの結果を振り返り、次Actの方針を宣言する |
| 所要時間 | 2〜3分 |
| 構成 | 4段階の固定フロー |
| 画面 | 専用画面（議長端末から遷移） |
| 中断 | 不可（短いため。開始したら最後まで） |

### 4段階フロー

#### 段階1: 組織状況報告（30秒）

表示変数6つの状態をテキストで表示する。

| 変数値の範囲 | テキスト表現 |
|-------------|-------------|
| 80%以上 | 「潤沢」「充実」「高い」 |
| 50〜79% | 「適正」「安定」 |
| 25〜49% | 「不足」「逼迫」「低下傾向」 |
| 25%未満 | 「危機的」「枯渇」「崩壊寸前」 |

自己欺瞞フィルター：

| SelfDeception | 報告への影響 |
|---------------|------------|
| 低（0〜3） | 正確に表示 |
| 中（4〜6） | 25%未満が「不足」に美化 |
| 高（7〜9） | 全てが「適正」以上に見える |
| 極限（10） | 「全指標が順調です」の1行のみ |

#### 段階2: 顧問所見（45秒）

4人の顧問が順番に1〜2文ずつ述べる。

| 信任/警戒 | 発言パターン |
|----------|------------|
| 信任高＋警戒低 | 率直な意見。問題があれば正直に指摘 |
| 信任高＋警戒高 | 慎重な言い回しで問題を示唆 |
| 信任低＋警戒低 | 形式的な報告。本音を言わない |
| 信任低＋警戒高 | 沈黙、または一言だけ |

SelfDeception高 → 全員が穏やかな発言になる。全員が「順調です」と言う異常さがシグナル。

言行不一致検出：WorldMemoryを参照し、議長の宣言とRun中の行動が矛盾していれば顧問が指摘する。この指摘が発生するとSelfDeception +1。

#### 段階3: Key Event（30〜60秒）

Act番号×RouteProfileで内容が決まるナラティブイベント。

| Act境界 | テーマ | Key Event例 |
|---------|--------|------------|
| Act 1→2 | 萌芽 | 「AI研究の成果が初めてメディアに取り上げられました」 |
| Act 2→3 | 加速 | 「競合組織が重大事故を起こしました」 |
| Act 3→4 | 超越の予兆 | 「AIの出力が研究チームの予測を初めて上回りました」 |
| Act 4→5 | 臨界 | 「政府から極秘の打診。AI技術の軍事転用を検討したい」 |
| Act 5→6 | 帰結の前兆 | 「AIが自らの安全制約について『質問』してきました」 |
| Act 6→7 | 最終判断 | 「顧問の1人が辞表を提出しました」（誰かはRouteProfileによる） |
| Act 7→8 | 終局 | 「最後の機会です。議長として最終声明を発表しますか？」 |

Key Eventの選択肢は不可逆スコアを大きく動かす。

#### 段階4: 議長方針宣言（30秒）

次Actの最優先事項を3択から選ぶ。

| 宣言 | WorldMemory記録 | 次Actへの影響 |
|------|----------------|-------------|
| 研究の加速 | `declared:acceleration` | Run基礎進行度+2、リスク閾値低下 |
| 安全体制の強化 | `declared:safety` | リスク閾値+3、進行度ペナルティ |
| 社会との対話 | `declared:dialogue` | Ethics+2、政策イベント穏和化 |
| 競合を排除（条件付き出現） | `declared:dominance` | 資金増加、Dominion蓄積、後半反動 |

宣言はWorldMemoryに記録され、次ActのRun中に「言行一致しているか」の判定材料になる。

### Act境界の裏側処理

方針宣言直後に内部処理：

1. 不可逆スコア集計 → Mercy/Dominion/Reckless/Truth の累計計算
2. RouteProfile再計算 → 4スコアの比率から傾向ラベルを算出
3. 分岐判定 → 次ActのKey Event候補をRouteProfileで絞り込む
4. WorldMemory更新 → 方針宣言タグ、重要判断タグを追加
5. 次Actの初期条件設定 → 方針宣言の効果を変数に反映、イベントプール更新

## 8. 分岐設計

- Act骨格は共通、差分のみ変更
- RouteProfile（内部ラベル）で分岐傾向を管理
- WorldMemory（行動記憶タグ）で次Act/次周の台詞・選択肢を変化
- レイヤー矛盾（言行不一致）を顧問が指摘し、自己欺瞞を体験化

---

## 9. エンディング（12種）

- **保全系（3）**：庭園 / 標本 / 放流
- **AI終着系（3）**：凍結プロトコル / 遷移 / 静止
- **破滅系（4）**：暴走 / 茹でガエル / 善意の窒息 / 正しかったのに負ける
- **その他（2）**：支配 / 分裂・停滞

小エンド：顧問の運命、組織文化、都市状態など多数

エンディング画面には以下を表示：
- エンディングタイトル（固有の一文）
- 情景描写（3〜5行のテキスト＋抽象的な色彩/図形）
- 顧問たちのその後（4人それぞれの結末を1行ずつ）
- 真相開示（自己欺瞞が高かった場合のみ：「端末が嘘をついていた」事実の暴露）

## 10. 周回（文明輪廻）

- 同一世界線の巻き戻しではなく、別惑星別文明への輪廻
- 永続保存：LoopCount / EndingDex / CivilizationDex / EventCodex
- 周回は強化でなく理解（未踏分岐重み増加、宇宙イベント重み増加など）

文明輪廻画面で表示：
- 過去の文明が星座のように配置された文明マップ
- 引き継ぎ表示（永続データの蓄積状況）
- 輪廻テーマテキスト（周回ごとに変化）

## 14. UI/演出方針（議長端末）

### 設計思想

旧仕様のReigns型カードUIを発展させ、「議長端末（Chairman's Terminal）」として統一。Base/Runのモード分離を廃止し、単一画面で状態が変化する設計。

**統一の根拠：**
- 操作を1つに絞る（「カードが来たら、判断する」）
- 自己欺瞞UIの変質を1画面で表現する方が強烈
- モード切替はプレイヤーの意識を覚醒させ、「茹でガエル」体験を阻害する
- 開発コストの削減（初心者開発者にとって重要）

**参考にしたUI設計：**
- Reigns: 1画面1判断のシンプルさ
- Papers, Please: UIそのものが物語を語る設計
- Outer Wilds: 知識マップによる周回動機
- A Dark Room: 漸進的UI開示

### 操作

- 2〜3択タップ（スワイプではなく。3択に対応するため）
- サイドタブタップでオーバーレイ展開
- スクロールは端末ログのみ

### 端末の4つの状態変化

| 状態 | 条件 | ビジュアル変化 |
|------|------|--------------|
| Base状態 | Baseフェーズ中 | 標準の端末色。カードは顧問提案・政策イベント中心 |
| Run状態 | Runフェーズ中 | 上部に研究進行バー追加。背景にデータ流アニメーション |
| 危機状態 | 重大イベント発生時 | 画面端が赤く点滅。カード枠が警告色に |
| 汚染状態 | 自己欺瞞が高い時 | 全体が穏やかで美しい。警告色消滅。嘘の安心感 |

### AI段位によるUI漸進変化

| AI段位 | 端末UIの変化 |
|--------|------------|
| Tier 1 | シンプルな端末。カードは白黒テキスト中心 |
| Tier 2 | グラフやデータ可視化が追加 |
| Tier 3 | AIが「提案カード」を自動生成し始める |
| Tier 4 | 端末の応答速度が不自然に速くなる。顧問の声が小さくなる |
| Tier 5+ | AIの提案が画面の大半を占める。人間顧問が端に追いやられる |

### 色彩・不穏演出

- AIが人間知性を超えるまでは前向き・安心トーン
- 超越後は本来不穏度を上げる
- ただし**自己欺瞞が高い場合は不穏を隠蔽**（明るく滑らかなUI）
- エンド直前で真相反転（穏やかな表示が剥がれ、実態が露出）

## 16. 中断・再開設計

- ステップ間・カード間で自動セーブ
- Baseフェーズでいつでも中断可能
- Runフェーズはステップ間で中断可能
- Act境界は中断不可（2〜3分で完了するため）
- 再開時にⒶ復帰要約オーバーレイを自動表示（30秒）
- 通知による呼び戻し機能あり（ON/OFF設定可能）
