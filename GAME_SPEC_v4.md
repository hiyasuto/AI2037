## ゲーム仕様書 v4.0（全議論統合版）

---

### 1. コンセプト

**一行要約：**  
「善意の組織がなぜ破滅するかを体験する、AI開発シミュレーション」

**テーマ：**  
AI開発の加速と、供給制約・制度・社会反応・倫理リスクの同時進行

**体験の核：**  
「自分は正しく選んだはずなのに、なぜこうなった」と思わせること

**プラットフォーム：** モバイル（買い切り）  
**1セッション：** 20–30分  
**1キャンペーン：** 6–8 Act  
**周回前提：** 2–4周で全体像が見える設計

---

### 2. ゲーム構造（2レイヤー＋政策イベント）

```text
Base（5分）
 ├─ 資源管理・人材配置・制度整備
 ├─ 政策イベント発生（Act中 5–8回、不定期）
 └─ Run出撃準備
       │
Run（12–15分）
 ├─ 研究進行（速度↔安全の逐次判断）
 ├─ 現場判断イベント
 └─ Run結果 → Baseに反映
       │
清算（1–2分）
 ├─ 資源変動・状態更新
 └─ 次セッションへ
       │
Act境界（自動）
 ├─ 不可逆スコア集計
 ├─ 分岐判定（RouteProfile再計算）
 ├─ Key Event / 顧問イベント
 └─ 次Actへ
```

**コアサイクル：** Base → Run → 清算 を繰り返し、Act境界で物語が分岐

---

### 3. 状態変数（合計12個）

#### 表示変数（UI表示、6個）

| 変数 | 役割 |
|------|------|
| 資金 | 全体の行動制約 |
| 電力 | 計算資源の前提条件 |
| 計算資源（マシン） | Run進行の燃料 |
| 人材 | 配置・効率・離職リスク |
| 制度 | 監査・法務・手続きの成熟度 |
| AI段位 | Tier表示。段位で解放・難度・不穏が変質 |

#### 非表示変数（数値非公開、2個）

| 変数 | 役割 |
|------|------|
| 倫理（Ethics） | 補償・透明性・弱者配慮の姿勢 |
| 自己欺瞞（SelfDeception） | 都合の悪い兆候を"見ない"組織文化 |

**非表示変数の可視化方針：**  
数値は出さない。代わりに**症状**で示す。

| 内部状態 | 症状表現 |
|----------|---------|
| 倫理低下 | ニューストーン悪化、外部抗議、顧問の沈黙 |
| 自己欺瞞上昇 | 会議ログの婉曲化、リスク文言の軟化、顧問が耳障りの良い報告しかしなくなる |
| 自己欺瞞高＋倫理低 | 新しい選択肢が生える（「却下する」「報告を書き換える」等。選択肢の存在自体が腐敗の証拠） |

#### 不可逆スコア（内部蓄積、4個）

| スコア | 蓄積条件 |
|--------|---------|
| Mercy | 補償・救済・弱者配慮の選択 |
| Dominion | 支配・独占・競合排除の選択 |
| Reckless | 速度優先・検証省略・リスク無視の選択 |
| Truth | 公開・透明性・内部告発容認の選択 |

**用途：** Act分岐判定、会話差分、イベント抽選重み、エンディング条件

---

### 4. Base（基地運営）

**目的：** ボトルネックを解き、Runの条件を整える

**操作：**
- 資源配分（資金→電力→計算資源のチェーン）
- 人材配置（研究/安全/制度/広報）
- 制度整備（投資すると短期は遅くなるが、後半の崩壊を防ぐ）

**設計原則：**
- 自動化を多めにし、手動管理で離脱させない
- 重要なのは「何に優先投資するか」の方針判断だけ

---

### 5. 政策イベント

**発生：** Base画面中にAct内で5–8回、不定期に発生  
**所要：** 30秒〜2分

**設計原則：**
- 単純2択ではなく、**状況説明＋2〜3択**
- 選択肢は過去の行動履歴・自己欺瞞レベルで**動的に増減**
- 各選択は必ず以下を動かす：
  - 表示変数 1–2個
  - 非表示変数 1–2個
  - 不可逆スコア 1個

---

### 6. Run（研究ローグライト）

**目的：** AI段位を進行させる。速度↔安全のトレードオフが中心

**所要：** 12–15分/回

**設計原則：**
- 選択肢を「カード」「ビルド」ではなく**「研究方針の逐次判断」**として表現
- Runの中の選択がBase/政策と同じ言語体系であること（別ゲーム化を防ぐ）

---

### 7. 分岐設計

- Act骨格は共通、差分のみ変更
- RouteProfile（内部ラベル）で分岐傾向を管理
- WorldMemory（行動記憶タグ）で次Act/次周の台詞・選択肢を変化
- レイヤー矛盾（言行不一致）を顧問が指摘し、自己欺瞞を体験化

---

### 8. エンディング（12種）

- **保全系（3）**：庭園 / 標本 / 放流
- **AI終着系（3）**：凍結プロトコル / 遷移 / 静止
- **破滅系（4）**：暴走 / 茹でガエル / 善意の窒息 / 正しかったのに負ける
- **その他（2）**：支配 / 分裂・停滞

小エンド：顧問の運命、組織文化、都市状態など多数

---

### 9. 周回（文明輪廻）

- 同一世界線の巻き戻しではなく、別惑星別文明への輪廻
- 永続保存：LoopCount / EndingDex / CivilizationDex / EventCodex
- 周回は強化でなく理解（未踏分岐重み増加、宇宙イベント重み増加など）

---

### 10. 宇宙探索（隠し要素）

- CosmicProgress（0–5）
- 5到達で宇宙イベント解禁
- 多惑星滅亡痕跡を通じて真相へ接続

---

### 11. 観測者（超常AI）

- 滅亡文明起源の自己進化AI
- 文明と進化AIを臨界経路データとして観測
- 介入手段は物理改変でなく情報・確率・誘導

---

### 12. キャラクター

- プレイヤー：議長（政策と方針の最終決定）
- 顧問4人：架空類型、信任/警戒の2軸で反応変化

---

### 13. UI/演出方針（Reigns型）

- **UIスタイル：** Reigns的カードUI（1画面1判断）
- **操作：** スワイプ or 2〜3択タップ
- **画面数：** 5（カード / Base / Run / Act境界 / エンディング）
- **ビジュアル：** ミニマルUI＋タイポ＋色彩推移

#### 色彩・不穏演出の核（重要）
- AIが人間知性を超えるまでは前向き・安心トーン
- 超越後は本来不穏度を上げる
- ただし**自己欺瞞が高い場合は不穏を隠蔽**（明るく滑らかなUI）
- エンド直前で真相反転（穏やかな表示が剥がれ、実態が露出）

→ 「同じ危機でも、自己欺瞞の高低で見える世界が変わる」UI設計

---

### 14. 中断・再開設計（モバイル必須）

- Run終了時自動セーブ
- Baseでいつでも中断
- 再開時30秒要約（直近判断、資源、AI段位）

---

### 15. 初期開発範囲（プロトタイプ）

- Act：2
- 政策イベント：16
- Run遭遇：10
- 顧問会話：20
- エンディング：3（放流 / 暴走 / 凍結）
- 周回引継ぎ：4要素
- 「語りたくなる瞬間」：1

**最優先検証：** WorldMemoryによる「記憶される選択」の体感成立
