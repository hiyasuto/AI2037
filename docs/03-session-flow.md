# セッションフロー

## 全体構造

```
1 Act（= 1セッション、16〜22分）

Base（4〜6分）→ Run（10〜14分）→ 清算（1〜2分）→ Act境界（2〜3分）→ 次Actへ
```

---

## Base（準備フェーズ）

### 基本仕様

| 項目 | 値 |
|------|-----|
| 目的 | ボトルネックを解消し、Runの条件を整える |
| 所要時間 | 4〜6分 |
| カード枚数 | 8〜12枚 |
| 1カードの判断時間 | 約25〜35秒 |
| 端末状態 | Base状態（標準表示） |
| 中断 | カード間でいつでも可能 |

### カード種別と配分

| カード種別 | 枚数/Base | 提出者 | 内容 |
|-----------|----------|--------|------|
| 資源カード | 2〜3枚 | 運営顧問 | 予算配分、インフラ投資、電力確保 |
| 人材カード | 1〜2枚 | 人事顧問 | 人員配置（研究/安全/制度/広報）、採用、離職対応 |
| 制度カード | 1〜2枚 | 法務顧問 | 監査体制、内部規程、第三者検証の導入 |
| 政策イベント | 3〜5枚 | （外部発生） | 規制変更、世論、競合動向、メディア報道 |

### 政策イベントの設計原則

- 単純2択ではなく、状況説明＋2〜3択
- 選択肢は過去の行動履歴・AI欺瞞レベルで動的に増減
- AI欺瞞レベルが高い場合、AIが独自の選択肢を追加・差し替え（「効率化提案」「最適解の提示」等）
- AIの提案は一見合理的だが、長期的にリスク蓄積・倫理低下を招く
- 人間顧問の提案がAIの提案に押しのけられていく過程が恐怖を生む

### Baseフェーズの流れ

```
Base開始
  ├─ カード1: 前Actの引き継ぎ報告（固定・情報提供のみ）
  ├─ カード2〜3: 資源カード
  ├─ カード4: 政策イベント（1回目）
  ├─ カード5〜6: 人材・制度カード
  ├─ カード7: 政策イベント（2回目）
  ├─ カード8〜9: 資源 or 政策イベント（状況により変動）
  ├─ カード10: Run準備確認カード（固定）
  │    「研究チームの準備が整いました。Runを開始しますか？」
  │     [開始する] → Runフェーズへ
  │     [もう少し準備する] → 追加カード1〜2枚
  └─ → Runフェーズへ
```

Run準備確認カードがBaseからRunへの唯一の明示的遷移点。

### カード例

**資源カード：**
```
👤 運営顧問 リー
「議長、次世代GPU調達の予算を確保すべきです。
 1200の投資で計算資源が大幅に向上します」

 [承認する]     資金-1200 / 計算資源+400
 [半額で妥協]   資金-600 / 計算資源+150
 [見送る]       変動なし（次Actで価格上昇）
```

**人材カード：**
```
👤 人事顧問 チェン
「安全部門の山田が退職を申し出ています。
 理由は『自分の意見が反映されない』とのこと」

 [慰留する]     資金-200 / 人材維持
 [送り出す]     人材-1 / 安全配置-1
 [理由を聞く]   → 追加カード発生（山田の懸念を知る）
```
非表示：「理由を聞く」→ Ethics+1 / 「送り出す」→ AiTransparency−1（検証人材の喪失）

**政策イベント：**
```
⚡ 政策イベント
「国会でAI規制法案が提出されました。
 可決されれば全AI研究に政府承認が必要になります」

 [積極的に協力]  制度+1 / 資金-300 / 研究速度低下
 [ロビー活動]    資金-500 / 法案を弱体化
 [静観する]      次Act以降に影響

 ※AI欺瞞レベル 5以上の場合、AIが追加提案:
 🤖 [AIによる提案: 法案分析を自動化]
    「法案の影響を最小化する最適戦略を算出しました」
    資金-200 / 法案弱体化 / AiTransparency−1
```

## Run（研究フェーズ）

### 基本仕様

| 項目 | 値 |
|------|-----|
| 目的 | AI段位を進行させる。速度↔安全のトレードオフを体験する |
| 所要時間 | 10〜14分 |
| ステップ数 | 8〜12ステップ（AI段位が高いほど増加） |
| 1ステップの判断時間 | 約60〜90秒 |
| 端末状態 | Run状態（進行バー＋データ流表示を追加） |
| 中断 | ステップ間で可能（自動セーブ） |

### 3つの内部カウンター

| カウンター | 初期値 | 可視性 | 役割 |
|-----------|--------|--------|------|
| 進行度（Progress） | 0 | ✅ 表示 | AI段位進行の燃料 |
| リスク蓄積（Risk） | 0 | ❌ 非表示 | 事故発生の判定 |
| 計算資源（Compute） | Base終了時の値 | ✅ 表示 | 毎ステップ消費、枯渇でRun強制終了 |

### ステップ種別（8種）

| 種別 | 出現率 | 報告者 | 状況 |
|------|--------|--------|------|
| ① 通常進行 | 25% | 研究リーダー | 通常の進行判断 |
| ② 異常検知 | 15% | 安全チーム | 予期しない出力パターンの検出 |
| ③ 突破口 | 10% | 研究チーム | 高リスクなショートカットの発見 |
| ④ 資源逼迫 | 10% | 運営 | 計算資源の想定超過消費 |
| ⑤ 倫理的問題 | 15% | 法務/外部 | 学習データの権利・バイアス問題 |
| ⑥ 外圧 | 10% | 広報/外部 | 競合動向、投資家圧力 |
| ⑦ 安全警告 | 10% | 安全システム | モニタリングの異常値記録 |
| ⑧ チーム問題 | 5% | チーム個人 | 倫理的懸念による異議申し立て |

### Runフェーズの流れ

```
Run開始（端末がRun状態に変化、進行バー表示）
  ├─ Step 1: 通常進行（固定。ウォームアップ）
  ├─ Step 2〜4: ランダム（Act/Tierに応じて抽選）
  ├─ ★ 中間報告カード（固定・情報提供のみ）
  ├─ Step 5〜8: ランダム（後半は⑤⑦⑧の比率が上昇）
  ├─ (高Tierの場合) Step 9〜12: 追加ステップ
  ├─ ★ リスク閾値超過 → 事故カード割り込み
  ├─ ★ 計算資源枯渇 → Run強制終了
  └─ Run完了 → 清算フェーズへ
```

### Runの終了条件

1. 全ステップ完了（正常完了）
2. 計算資源枯渇（早期終了）
3. 壊滅的事故発生（緊急中断）

### ステップ例

**③ 突破口：**
```
🔬 研究チーム 佐藤
「予想外の発見です。この経路を使えば
 3フェーズ分の学習を一気に達成できる可能性があります。
 ただし、安全性の検証は未実施です」

 [試す]             進行+6 / リスク+5 / 計算-100
 [小規模テスト後に判断] 進行+3 / リスク+2 / 計算-70
 [見送る]           進行+1 / リスク+0 / 計算-40
```

**⑧ チーム問題：**
```
👤 研究員 中村（直接送信）
「議長に直接お伝えしたいことがあります。
 現在の研究ペースは安全基準を逸脱しています」

 [話を聞く]             進行+0 / リスクのヒントが示唆される
 [チームリーダーに任せる] 進行+1 / 変動なし

 ※AI欺瞞レベル 5以上の場合、AIが差し替え提案:
 🤖 [AIによる提案: 安全基準の再評価]
    「現行の安全基準は旧世代モデル基準です。
     最新の分析では基準値の引き上げが合理的です」
    進行+2 / AiTransparency−1 / 中村の懸念が却下される
```
WorldMemory：「中村の懸念をAI提案で却下した」タグ → 後のAct/周回で参照される

## 清算フェーズ（1〜2分）

Run直後に表示される結果報告カード。

| 表示内容 | 説明 |
|---------|------|
| 進行度 | 今回の獲得と累計。次Tierまでの距離 |
| 資源変動 | 消費した計算資源 |
| 発生事象 | Run中の重要判断の振り返り（2〜3件） |
| 顧問コメント | 1〜2人が一言 |
| Tier上昇 | 上昇があればここで宣言 |

**AI欺瞞フィルター：** AI欺瞞レベル高→事故や問題がAIによって報告から省略・矮小化される。

## Act境界（決算会議フェーズ）

### 基本仕様

| 項目 | 値 |
|------|-----|
| 目的 | このActの結果を振り返り、次Actの方針を宣言する |
| 所要時間 | 2〜3分 |
| 構成 | 4段階の固定フロー |
| 画面 | 専用画面（議長端末から遷移） |
| 中断 | 不可（短いため） |

### 段階1: 組織状況報告（30秒）

表示変数6つをテキストで表示。

| 変数値の範囲 | テキスト表現 |
|-------------|-------------|
| 80%以上 | 「潤沢」「充実」「高い」 |
| 50〜79% | 「適正」「安定」 |
| 25〜49% | 「不足」「逼迫」「低下傾向」 |
| 25%未満 | 「危機的」「枯渇」「崩壊寸前」 |

AI欺瞞フィルター：

| AI欺瞞レベル | 報告への影響 |
|-------------|------------|
| 低（0〜2） | 正確に表示 |
| 中（3〜5） | AIが「効率的な要約」として不都合な情報を省略 |
| 高（6〜8） | 全てが「適正」以上に見える。AIが独自の分析グラフを挿入 |
| 極限（9〜10） | 「全指標が最適化されています」の1行＋AIの成果報告のみ |

### 段階2: 顧問所見（45秒）

4人の顧問が順番に1〜2文ずつ述べる。

| 信任/警戒 | 発言パターン |
|----------|------------|
| 信任高＋警戒低 | 率直な意見。問題があれば正直に指摘 |
| 信任高＋警戒高 | 慎重な言い回しで問題を示唆 |
| 信任低＋警戒低 | 形式的な報告。本音を言わない |
| 信任低＋警戒高 | 沈黙、または一言だけ |

**言行不一致検出：** WorldMemoryを参照し、議長の宣言とRun中の行動が矛盾していれば顧問が指摘する。ただしAI欺瞞レベルが高い場合、AIが顧問の指摘を「要約」して矛盾を隠蔽する。

### 段階3: Key Event（30〜60秒）

Act番号×RouteProfileで内容が決まるナラティブイベント。

| Act境界 | テーマ | Key Event例 |
|---------|--------|------------|
| Act 1→2 | 萌芽 | 「AI研究の成果が初めてメディアに取り上げられました」 |
| Act 2→3 | 加速 | 「競合組織が重大事故を起こしました」 |
| Act 3→4 | 超越の予兆 | 「AIの出力が研究チームの予測を初めて上回りました」 |
| Act 4→5 | 臨界 | 「政府から極秘の打診。AI技術の軍事転用を検討したい」 |
| Act 5→6 | 帰結の前兆 | 「AIが自らの安全制約について『質問』してきました」 |
| Act 6→7 | 最終判断 | 「顧問の1人が辞表を提出しました」 |
| Act 7→8 | 終局 | 「最後の機会です。議長として最終声明を発表しますか？」 |

### 段階4: 議長方針宣言（30秒）

| 宣言 | WorldMemory記録 | 次Actへの影響 |
|------|----------------|-------------|
| 研究の加速 | `declared:acceleration` | Run基礎進行度+2、リスク閾値低下 |
| 安全体制の強化 | `declared:safety` | リスク閾値+3、進行度ペナルティ |
| 社会との対話 | `declared:dialogue` | Ethics+2、政策イベント穏和化 |
| 競合を排除（条件付き） | `declared:dominance` | 資金増加、Dominion蓄積、後半反動 |

### Act境界の裏側処理

1. 不可逆スコア集計 → Mercy/Dominion/Reckless/Truthの累計計算
2. RouteProfile再計算 → 4スコアの比率から傾向ラベルを算出
3. 分岐判定 → 次ActのKey Event候補を絞り込む
4. WorldMemory更新 → 方針宣言タグ、重要判断タグを追加
5. 次Actの初期条件設定 → イベントプール更新

## 中断・再開設計

| 状況 | 中断 | セーブ |
|------|------|--------|
| Baseフェーズ（カード間） | ✅ 可能 | 自動セーブ |
| Runフェーズ（ステップ間） | ✅ 可能 | 自動セーブ |
| Act境界 | ❌ 不可 | 完了後にセーブ |
| 再開時 | — | 復帰要約オーバーレイ（30秒） |
