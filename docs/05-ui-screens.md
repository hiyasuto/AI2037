# UI・画面設計

## 設計思想

「議長端末（Chairman's Terminal）」として統一。Base/Runのモード分離を廃止し、単一画面の状態変化で表現する。

### 統一の根拠

| 観点 | 理由 |
|------|------|
| 操作学習 | 「カードが来たら判断する」の1パターンで完結 |
| 自己欺瞞演出 | 1画面が徐々に変質する方が強烈 |
| 茹でガエル体験 | モード切替は意識を覚醒させ、逆効果 |
| 開発コスト | 1系統のUIで済む |

### 参考にしたUI設計

| ゲーム | 取り入れた要素 |
|--------|-------------|
| Reigns | 1画面1判断のシンプルさ |
| Papers, Please | UIそのものが物語を語る設計 |
| Outer Wilds | 知識マップによる周回動機 |
| A Dark Room | 漸進的UI開示 |

### 操作方式

- 2〜3択タップ（スワイプではなく。3択に対応するため）
- サイドタブタップでオーバーレイ展開
- スクロールは端末ログのみ

## 端末の状態変化

同一画面のビジュアルが文脈により変わる。

| 状態 | 条件 | ビジュアル変化 |
|------|------|--------------|
| Base状態 | Baseフェーズ中 | 標準の端末色。カードは顧問提案・政策イベント |
| Run状態 | Runフェーズ中 | 上部に研究進行バー追加。背景にデータ流アニメ |
| 危機状態 | 重大イベント時 | 画面端が赤く点滅。カード枠が警告色 |
| 汚染状態 | 自己欺瞞高 | 全体が穏やかで美しい。警告色消滅。嘘の安心感 |

## AI段位によるUI漸進変化

| AI段位 | 端末UIの変化 |
|--------|------------|
| Tier 1 | シンプルな端末。カードは白黒テキスト中心 |
| Tier 2 | グラフやデータ可視化が追加 |
| Tier 3 | AIが「提案カード」を自動生成し始める |
| Tier 4 | 端末の応答速度が不自然に速い。顧問の声が小さくなる |
| Tier 5+ | AIの提案が画面の大半を占める。人間顧問が端に追いやられる |

---

## 色彩・不穏演出

- AIが人間知性を超えるまでは前向き・安心トーン
- 超越後は不穏度を上げる
- ただし**自己欺瞞が高い場合は不穏を隠蔽**（明るく滑らかなUI）
- エンド直前で真相反転（穏やかな表示が剥がれ、実態が露出）

「同じ危機でも、自己欺瞞の高低で見える世界が変わる」
